비기능 요구사항 (시그 책임)

서버·인프라는 비쥬웍스가 운영하므로, 시그가 만든 애플리케이션과 인수인계 품질에 대한 요구만 담았습니다. 자체 운영이라 인수인계·유지보수성은 오히려 더 중요합니다.

01요구 기준

애플리케이션 · 인수인계 중심
항목요구 기준담당
성능Core Web Vitals — LCP ≤2.5s · INP ≤200ms · CLS ≤0.1 (모바일/데스크톱, 실사용 75백분위) · 프런트 렌더·캐시·번들·중복요청 방지 · 통합 부하테스트 참여시그
보안OWASP ASVS 5.0 Level 2 (프런트·CMS·BFF·파일 업로드·OAuth) · 오픈 전 진단→조치→재진단 · Critical·High 0건 · 권한 없는 접근 서버단 401/403 검증시그
접근성KWCAG 2.2 · 자동 + 전문가 수동 검수 · 결과서·조치내역 제출시그
품질 · 호환디바이스·브라우저 QA 매트릭스 · 지원 브라우저·최소 버전 명시시그
인수인계 · 유지보수성 ⭐소스코드 + Git 변경 이력 · 배포 런북·절차 문서 · 환경 재구성 문서 · 운영자 매뉴얼·인수 교육 · 측정 계획서(FR-13)시그
하자보수무상 하자보수 기간·범위 명시 (오픈일·최종검수일 중 늦은 날 다음날부터)시그
⭐ 왜 인수인계가 핵심인가 오픈 후 운영·유지보수는 비쥬웍스가 직접 합니다(시그는 인수인계 지원만). 소스·배포 런북·환경 문서·운영자 매뉴얼이 부실하면 우리가 사이트를 못 굴립니다. 그래서 자체 운영일수록 이 항목이 더 중요합니다.

02범위에서 제외 (비쥬웍스 몫)

서버·인프라는 우리가 운영하므로 시그에 요구하지 않음
  • 서버 가용성·용량 · 인프라 백업(RPO/RTO) · OS·네트워크 보안 패치 · DB 성능 튜닝 · 배포 실행.
  • 단, 시그는 백업 대상 목록·복구 절차·환경 재구성 문서는 인수인계 산출물로 제공(위 표 참조).
