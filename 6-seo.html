05 · SEO Assets

SEO 실측 자산

레거시 소스에서 직접 추출한 SEO 현황입니다. 이관 시 살릴 자산과 반드시 고쳐야 할 문제를 구분했습니다. FR-10 SEO 관리 · FR-07 다국어 연계.

01메타 · 태그 구조

페이지별 동적 세팅 — 이관 시 seo_meta로 승계
항목현황이관
title · description · keywords페이지 레코드에서 동적 세팅 (page_name→title · info01→description · info02→keywords)승계 가능
canonical있음 · PC/모바일 alternate(min-width 640) 병행 · /m/m/ 중복 보정 흔적승계 가능
Open Graphog:type(article)·title·url·image·site_name·description·updated_time 완비승계 가능
Twitter Cardsummary · title·description·image승계 가능
검색엔진 인증소스에 실제 값 없음 → 계정측(GSC·네이버)마케팅팀

→ 레거시가 이미 페이지별로 SEO 문구를 DB에 보관하고 있어, FR-10의 seo_meta(page × language) 구조로 자연스럽게 이관됩니다.

02구조화 데이터 (schema.org)

이미 보유한 자산 — 재활용 가치 높음
유형현황
FAQ (FAQPage)안질환 9개 페이지에 JSON-LD FAQ 존재 (백내장·녹내장·노안·당뇨망막병증·황반변성·비문증·망막정맥/동맥폐쇄·라식라섹)
블로그 (Yoast)WordPress SEO(Yoast) 자동 스키마 — 블로그 이전 시 재구성 필요
재활용 포인트 기존 FAQ 구조화 데이터는 FR-10 기본 구조화(필수) 및 FR-12 GEO(선택)에서 AI 검색·리치결과용 자산으로 바로 재사용할 수 있습니다.

03sitemap · robots

현행 상태 — 오픈 전 반드시 교체
파일현황판정
sitemap.xml79 URL · lastmod 2020-03-27(6년 경과) · 도메인 bseye.kr(구 도메인) · http · 구 URL 구조 · 무료툴 수동생성폐기·재생성
robots.txtUser-agent: * / Disallow: / — 전체 크롤링 차단확인·교체
오픈 차단 위험
  • robots.txt가 전체 차단(Disallow: /) — 이 백업이 스테이징본일 수 있으니 라이브 robots.txt 1회 확인. 신규 오픈 시 이 설정이 그대로 가면 색인 전멸.
  • sitemap이 6년 전·구 도메인 기준 — 신규는 언어별 자동 생성 sitemap으로 새로 만들어야 함(FR-07).

04다국어 SEO

현행은 분산 — FR-07에서 통합
  • hreflang 없음 — 언어별(en·zh·ja·vi) 서브도메인이 물리적으로 분리돼 있어 대체 언어 표시가 없음.
  • → 검색엔진이 언어판을 연결하지 못하는 상태. FR-07에서 경로 기반 locale + hreflang 자동으로 해소.

05이관 주의 (체크리스트)

SEO 자산 보존 + 위험 제거
  • robots.txt — 오픈 시 전체 차단 해제 확인 (가장 위험).
  • sitemap — 신규 도메인·URL·언어별로 자동 생성 재작성 (구 sitemap 폐기).
  • 301 redirect — 구 URL(110개 × PC/모바일) → 신규 전량 매핑 (FR-01).
  • 메타 승계 — 페이지별 title/description/keywords를 seo_meta로 이관 (FR-10).
  • 구조화 데이터 — FAQ 9종 재활용, 기본 4종 자동생성 (FR-10) + GEO 확장은 선택 (FR-12).
  • 검색엔진 인증·소유권 — GSC·네이버 소유권을 당사 명의로 이전 (마케팅팀).
